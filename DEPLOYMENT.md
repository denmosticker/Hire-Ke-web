# HireKe Free Demo Deployment

This setup is for a free Render demo. It runs the Express app and static pages from one Node service.

## What is free-demo safe

- The app boots with `npm start`.
- Render provides the public `onrender.com` URL through `RENDER_EXTERNAL_URL`.
- `JWT_SECRET` is generated by Render from `render.yaml`.
- Payments are mocked with `PALPLUS_MOCK=true`.

## Important limits

- SQLite is stored at `/tmp/hireke.db`, so data can be lost after restarts or redeploys.
- Uploaded files use local storage, so uploads can also be lost.
- Free Render services can spin down after inactivity, so first load may be slow.

## Deploy on Render

1. Push this folder to a GitHub repository.
2. In Render, create a new Blueprint from the repository.
3. Render will read `render.yaml` and create the free web service.
4. After the first deploy, open:

```text
https://your-service-name.onrender.com
```

5. Check the health endpoint:

```text
https://your-service-name.onrender.com/api/health
```

## Optional environment variables

Add these in Render only when you want the real integrations:

```text
SMTP_HOST
SMTP_PORT
SMTP_USER
SMTP_PASS
SMTP_SECURE
EMAILJS_SERVICE_ID
EMAILJS_TEMPLATE_ID
EMAILJS_PUBLIC_KEY
EMAILJS_PRIVATE_KEY
PALPLUS_STK_PUSH_URL
PALPLUS_API_KEY
PALPLUS_CLIENT_ID
PALPLUS_SECRET_KEY
PALPLUS_CALLBACK_SECRET
PALPLUS_CALLBACK_URL
PALPLUS_REQUIRE_SIGNATURE
```

When moving beyond the free demo, replace SQLite/local uploads with Postgres and object storage.
